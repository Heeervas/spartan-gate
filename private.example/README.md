# Private Overrides

Copy this directory to `private/` for machine-specific configuration:

```sh
cp -R private.example private
mv private/env/local.env.example private/env/local.env
```

`private/` is ignored by Git. Keep real tokens, host paths, Codex profiles, local ports, and personal notes there.

Use `compose.local.yml` for overrides that should not be public. The public Compose already marks fields as `required`, `recommended`, or `private/local` so it is clear what belongs here.

Extra Caddy listeners generated by `scripts/add-port.sh` live in `private/caddy.local.d/`.

Private outbound domains live in `private/outbound-proxy/whitelist.private.txt`.
The helper `scripts/whitelist-domain.sh` writes there by default.

Private runbooks and local decisions belong in `private/docs/`. Use `private.example/docs/README.md` as the template.

Use isolated data paths by default. Only point these variables at old stack data when you are intentionally migrating:

- `SPARTAN_HERMES_DATA_PATH`
- `SPARTAN_CLAWROUTE_DATA_PATH`
- `SPARTAN_GOGCLI_DATA_PATH`
